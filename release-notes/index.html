SAP S/4HANA Cloud SDK – Release Notes

Version 1.8.0

February 1, 2018 - Maven Central

Known Issues:
  • Using version 1.8.0 is NOT recommended for a multi-tenant application in the Neo environment of SAP Cloud Platform: This version has a bug that manifests only in the Neo environment of SAP Cloud Platform where running in non-container managed threads would always use the tenant context of the provider, not the consumer account. We plan to fix this bug in the next release. Applications running in the Cloud Foundry environment are not affected by this bug.

Compatibility Notes:
  • Some purely JAX-RS related classes have been moved from the com.sap.cloud.s4hana.frameworks:cxf module to a new JAX-RS framework module named com.sap.cloud.s4hana.frameworks:jaxrs. Some classes have been moved accordingly:
    • GsonMessageBodyProvider from package com.sap.cloud.sdk.frameworks.cxf.jaxrs to com.sap.cloud.sdk.frameworks.jaxrs,
    • JaxRsErrorResponseBuilder and JaxRsExceptionMapper from com.sap.cloud.sdk.frameworks.cxf.jaxrs to com.sap.cloud.sdk.frameworks.jaxrs.response, and
    • NotFoundExceptionMapper and WebApplicationExceptionMapper from com.sap.cloud.sdk.frameworks.cxf.servlet.response.mapper to com.sap.cloud.sdk.frameworks.jaxrs.response.mapper.
  • The following exception classes have been renamed:
    • TenantNotFoundException to TenantNotAvailableException,
    • KeyStoreException to KeyStoreNotFoundException, and
    • SecretStoreException to SecretStoreNotFoundException.

Improvements

  • Introduce TenantAccessException and UserAccessException for a cleaner contract on the tenant and user accessors and facades; rename TenantNotFoundException to TenantNotAvailableException.
  • Introduce KeyStoreAccessException and SecretStoreAccessException for a cleaner contract on the secret store accessor and facade; rename KeyStoreException to KeyStoreNotFoundException and SecretStoreException to SecretStoreNotFoundException.
  • Improve Javadoc in cloudplatform.* packages.

Fixed Issues

  • Remove tomcat-catalina lib from TomEE Maven plugin configuration in application/pom.xml generated by scp-cf-tomee archetype in order to fix local deployment issues (e.g., java.lang.NoSuchMethodError: org.apache.tomcat.util.ExceptionUtils.preload()V) on some developer machines.
  • Introduce additional RequestContextListeners for SAP Cloud Platform, Neo to avoid failing JNDI lookups in non-container managed threads.
  • Fix issue in HystrixUtil.getUserIsolatedKey not considering tenant context (this may have caused circuit breakers to trigger for users with the same name across different tenants).
  • Fix issue with multiple class loaders in FacadeLocator.

Version 1.7.1

January 18, 2017 - Maven Central

Compatibility Notes:
  • The update of the TomEE adapters for Arquillian to version 1.7.5 makes the use of the dependency org.jboss.arquillian.junit:arquillian-junit-container obsolete. This dependency has been removed from the SDK BOM, you may have to either remove the dependency from your integration-tests/pom.xml file or add a version to it (e.g., 1.1.5.Final).
  • Due to the switch from MockServer to WireMock, any tests written using the MockServer implementation have to be migrated (see Getting Started with WireMock).
  • DestinationType is now an enum instead of a class to better reflect the available destination types.

New Functionality

  • Add new archetype com.sap.cloud.s4hana.archetypes:scp-neo-javaee7 for Java EE 7 on SAP Cloud Platform Neo.
  • Support for on-premise connectivity via SAP Cloud Connector on SAP Cloud Platform Cloud Foundry.
  • Provide API about the state of feature toggles via FeatureToggleServlet.
  • Onboard and offboard tenants for schema-based multi-tenancy with SchemaPerTenantProvisioner for liquibase.
  • Add a new class LogInputSanitizer that can be used to prevent log forging.
    • The LogInputSanitizer comes with an END_OF_LOG token (by default: (END OF LOG ENTRY)) that will be appended to the end of each log entry.
    • If an attacker tries to inject log entries using this token, the sanitizer will catch it and replace the forged token by a warning token.

Improvements

  • The destination cache on SAP Cloud Platform Cloud Foundry is now only invalidated if destinations were retrieved successfully. Otherwise, the cache remains valid.
  • DestinationType, AuthenticationType, and ProxyType now provide methods for handling conversion from and to String identifiers.
  • Add additional AuthenticationType enum values which are available on SAP Cloud Platform Neo and/or Cloud Foundry.
  • Refine Destination interface: improve contract on some methods that may throw a DestinationAccessException and add method getDescription().
  • Add newTenantIsolatedOrGlobalKey to CacheKey.
  • Add getTenantIsolatedOrGlobalKey to HystrixUtil.
  • Move helper classes for managing concurrency on SAP Cloud Platform Neo to module com.sap.cloud.s4hana.cloudplatform:concurrency-scp-neo.
  • Replace MockServer with WireMock for MockUtil.mockErpServer and MockUtil.mockServer.
    • Instead of returning a ClientAndServer object, the affected MockUtil methods now return a WireMockRule.
    • This allows you to use the JUnit rule annotation for easier setup & teardown of the mock server.
  • Update version of TomEE adapters for Arquillian to 1.7.5 (from 1.7.4).
  • Update Jackson to version 2.9.3 (from 2.8.7).

Fixed Issues

  • OData virtual data model (VDM) for S/4HANA: When using function imports, the VDM now correctly retrieves a CSRF token before executing the request.
  • The toString method of destinations now replaces sensitive information such as passwords with the String (hidden).

Version 1.6.0

December 21, 2017 - Maven Central

Compatibility Notes:
  • Builder methods for navigation properties have been renamed from to[NavigationProperty] to [navigationProperty] in order to improve readability.
  • Methods of the VDM such as select or filter that before expected a generic EntityField instance now expect entity type-specific instances, for example, BusinessPartnerField (only relevant if you used your own EntityField instances, for example, to represent custom fields).
  • Previously malformed acronyms in field names have been fixed, resulting in changed names of variables available from entity types (example: BusinessPartner.BUSINESS_PARTNER_U_U_ID has been renamed to BusinessPartner.BUSINESS_PARTNER_UUID).

New Functionality

  • Support type-safe expand for OData queries with the S/4HANA Virtual Data Model (VDM).
    • The expand functionality is integrated into the select method of fluent helpers. Please note that there is no separate expand method.
    • The fields to select from the expanded entities can be specified in the same manner.
    • Example with the business partner service: use 
      getAllBusinessPartner().select(
  BusinessPartner.TO_BUSINESS_PARTNER_ROLE,
  BusinessPartner.TO_BUSINESS_PARTNER_ADDRESS.select(BusinessPartnerAddress.COUNTRY)
)
      to retrieve all business partners with their roles (all fields of role) and the country field of their addresses.
  • SAP Cloud Platform Cloud Foundry: Retrieve destinations including credentials from destination service (beta) and support basic authentication flows via the Cloud Platform abstractions that were already available on SAP Cloud Platform Neo.

Improvements

  • Improve usability of builder methods for navigation properties of VDM entities. Use, for example, BusinessPartner.builder().businessPartnerAddress(...) to add addresses while constructing an entity.
  • The methods for modifying navigation properties, set[NavigationProperty] and add[NavigationProperty], no longer attempt to fetch the associated entities first.
  • Add method to VDM objects to set a custom field using an EntityField object and a type-safe value.
  • Improve Javadoc for classes dealing with BAPI and RFC queries (see packages com.sap.cloud.sdk.s4hana.connectivity.rfc and com.sap.cloud.sdk.result).
  • Rename method doCommit in BapiQuery and RfcQuery to isPerformingTransactionalCommit. doCommit is now deprecated.
  • Retrieve generic destinations (in addition to RFC and HTTP) from the Destination Facade on both Neo and Cloud Foundry, introduce MAIL and LDAP as additional destination types.
  • Add best effort isolated cache key to CacheKey and a method newEquallyIsolatedKey to retrieve a cache key with same level of isolation as a given cache key.
  • Add best effort isolated command key to HystrixUtil, used as default by CachingCommand.

Fixed Issues

  • Fix compiler warning about unsafe varargs for select methods in the VDM.
  • Fix malformed acronym names for fields in the VDM.
  • Fix bug with transaction handling for BAPI queries to SAP S/4HANA on premise.
  • Fix failing deserialization of date and time fields into LocalDate and LocalTime when accessing BAPI and RFM results.
  • Fix issues with audit logging in AuditLoggingStateRepository (Togglz integration).
  • Fix some issues where input streams were not closed properly before.
  • Rename AutorizationApiDestination to AuthorizationApiDestination.

Version 1.5.0

December 7, 2017 - Maven Central

Compatibility Notes:
  • The methods for handling custom fields introduced in version 1.3.0 have been renamed from getField to getCustomField (hasCustomField, getCustomFieldNames) in order to clearly indicate that they deal with custom fields (i.e., extensions in S/4HANA).

New Functionality

  • Provide navigation properties as part of OData virtual data model (VDM) for S/4HANA.
    • Retrieve values of navigation properties of an entity via get[NavigationProperty]OrNull and get[NavigationProperty]OrFetch (examples: BusinessPartner.getBusinessPartnerAddressOrFetch and BusinessPartner.getBusinessPartnerAddressOrNull).
    • Modify navigation property values via set[NavigationProperty] (to overwrite) and, in case of 1:n associations, add[NavigationProperty] (examples: BusinessPartner.setBusinessPartnerAddress and BusinessPartner.addBusinessPartnerAddress).
  • Support deep insert via VDM by leveraging navigation properties.
  • Allow to modify values of custom fields.
    • Set the value of a custom field of an entity using setCustomField.
    • Custom fields of entities thus modified are sent to S/4HANA as part of create or update operations and persisted (if the custom field exists in the S/4HANA Cloud system).

Improvements

  • The ODataVdmErrorResultHandler used by default in all VDM operations exposes more information about exceptions that occur during calls of the VDM.
  • Method getCustomFields returns a map of all custom fields of an entity and their values.

Fixed Issues

  • Adjust default memory provisioned for applications generated by scp-cf-tomee, scp-cf-tomcat, or scp-cf-spring archetypes to ensure sufficient memory and metaspace.

Version 1.4.0

November 23, 2017 - Maven Central

Compatibility Notes:
  • As part of the update to the OData virtual data model (VDM) for SAP S/4HANA Cloud 1711, the name or package of some service, entity and helper classes has changed. The Javadoc of the VDM documents the new names.

New Functionality

  • This version updates the OData virtual data model to the newest release 1711 of SAP S/4HANA Cloud. This includes:
    • Almost 40 completely new services (available from package com.sap.cloud.sdk.s4hana.datamodel.odata.services),
    • More than 300 new operations,
    • More than 100 new entity types.
    The SDK supports all OData services listed in the SAP API Business Hub for SAP S/4HANA Cloud.
  • Allow to transparently use an additional destination for RFC connections to a system that is already connected via HTTP. By default, this destination is expected to have the suffix _RFC, for example, ErpQueryEndpoint_RFC. See the explanation of the different options for the ErpConfigContext.

Improvements

  • Expand existing virtual data model with new operations and properties (see above).
  • Improve Javadoc of existing virtual data model classes based on information from SAP API Business Hub.
  • Provide additional user and tenant information on SAP Cloud Platform, Cloud Foundry.
  • Maven archetypes for Cloud Foundry use Java version 8 by default.

Fixed Issues

  • Serialize boolean properties according to OData specification.
  • Increase timeout of cx-server script.
  • Select methods on virtual data model fluent helpers can be mocked.

Version 1.3.0

November 9, 2017 - Maven Central

Compatibility Notes:
  • Static references to virtual data model service classes are no longer supported. To migrate, instantiate classes with new or inject them if using a JSR 330 compliant framework. This simplifies testing and extensibility. Each service class provides a default implementation starting with prefix "Default".
  • Setter methods on entity objects return void instead of the instance itself. Use the builder() methods instead.

New Functionality

  • OData virtual data model allows creating, updating, and deleting entities. The VDM service classes are enhanced by additional methods (e.g. createBusinessPartner in class BusinessPartnerService).
  • OData virtual data model receives two new execute methods:
    • Method execute with an ErpEndpoint argument
    • Method execute without any parameter. In this case the default ErpConfigContext is considered.
  • Introduce builder pattern for OData virtual data model which simplifies instance creation leveraging the fluent pattern.
  • OData virtual data model allows custom field extensions for read access. If an OData service returns entities with additional values, they can be read by accessing the getField method of the derived entity class. hasField and getFieldNames can be used to determine which fields are available.

Improvements

  • Virtual data model service classes can now be instantiated using dependency injection. This simplifies testing and mocking, allowing tests to be independent from an S/4HANA downstream system.
  • Setter methods on entity objects return void instead of instance. Use the respective builder() methods instead.
  • Improve Javadoc of ErpEndpoint and ErpConfigContext.

Fixed Issues

  • Fix long startup time of local Tomcat deployment with Jetty.

Version 1.2.0

October 26, 2017 - Maven Central

Compatibility Notes:
  • This version removes all starter parent modules, including their use in the scp-neo-javaee6 archetype. It is strongly encouraged for Maven projects using starter parents to migrate to explicit configuration of properties and plugins.
  • This version adds the argument trustStorePassword to mockDestination() and mockErpDestination(). To migrate to these methods, set trustStorePassword to null where ever previous versions of these methods were called.

Deprecation Note: This version deprecates the obsolete functionality of retrieving ErpSystemInfo.

Known Issues

  • The scp-neo-maven-plugin in the scp-neo-javaee6 archetype uses a non-existing version. Please use version 1.2.0 instead of the declared version 1.2.0-SNAPSHOT when generating a new project from this archetype.

New Functionality

  • Add getTrustStorePassword() to Destination.
  • Add newUserOrTenantIsolatedKey() to CacheKey for a more graceful construction.
  • Add getUserOrTenantIsolatedKey() to HystrixUtil to more gracefully obtain a Command key.

Improvements

  • Add Javadoc links to the SAP API Business Hub in the OData virtual data model.
  • Add Javadoc for withErrorHandler() and toQuery() methods in the OData virtual data model.
  • Add mockErpServer() method with destinationName parameter.
  • Make the use of SapClient optional in mockErpServer() methods.
  • Improve the throws specification of several methods in CacheKey.
  • Improve exception handling for SOAP-based remote function module invocation.
  • Improve key handling with constructors using classes with suffix ByKeyFluentHelper.
  • Remove final on some classes (e.g., ErpEndpoint) to enable easier mocking.
  • Remove JSON building duration in QueryExecutionMeasurements to be measured as request building duration.

Fixed Issues

  • Fix ODataVdmErrorResultHandler to extend DefaultErrorResultHandler instead of implementing ErrorResultHandler.
  • Fix mockErpDestination() adding a sap-client header which ignores the SapClient in the ErpConfigContext.

Version 1.1.2

October 12, 2017 - Maven Central

Compatibility Note: This version uses Mockito 2.10.0, Mockito 1.x is no longer supported.

Details on the incompatible changes can be found here.

New Functionality

  • Add toQuery() method to all service classes of the S/4HANA virtual data model for OData.
    This method generates a service's underlying ODataQuery without executing it.
  • Add ODataVdmErrorResultHandler to provide more meaningful OData error messages.
  • Add default index.html to web application root of all archetypes.

Improvements

  • Extend Javadoc for public API of SDK.
  • Add Javadoc for S/4HANA virtual data model based on the SAP API Business Hub.
  • Add explicit Java compiler source and target version 1.7 to Maven archetypes.
  • Add explicit finalName in web descriptor of scp-neo-javaee6 archetype.

Fixed Issues

  • Fix shutdown of ConnectionManager in DestinationsRequestContextListener.
  • Fix building of composite keys in getByKey() methods of the S/4HANA virtual data model for OData.
  • Fix missing Javadoc creation for Lombok-generated code.
  • Fix unused but declared dependencies.
  • Fix used but undeclared dependencies.

Version 1.1.1

September 29, 2017 - Maven Central

Compatibility Note: OData navigation properties in S/4HANA virtual data model now use prefix fetch instead of get.

For example, a property that was previously named getItem() is now called fetchItem().

New Functionality

  • Support of SAP Java Connector API 3.0.17 for RFC on-premise connectivity to S/4HANA.
  • Add DestinationType to represent different destination types.
  • Add GenericDestination as generic base interface for all destination types.
  • Add RfcDestination to represent RFC destinations.
  • Add additional methods to DestinationAccessor for additional destination types.
  • Support additional properties within the destinations environment variable. For details, see ScpCfDestinationFacade.
  • Support mvn tomee:run and mvn jetty:run in TomEE and Tomcat archetypes for Cloud Foundry.

Improvements

  • DestinationAccessor now throws DestinationOAuthTokenException for missing OAuth tokens.
  • MockUtil now throws TestConfigurationError instead of ShouldNotHappenException for test configuration issues.
  • MockUtil now redirects audit logs to the default test log instead of ignoring them.
  • Extend Javadoc of S/4HANA virtual data model.

Fixed Issues

  • Fix Lombok using scope compile instead of provided.
  • Fix issue with local containers not working due to missing audit logger dependency.
